Instruções para corrigir a <img> que não aparece na interface

### 1. Confirme que o `<img>` efetivamente tem tamanho
- Verifique no DevTools (Inspector) se o elemento `<img>` possui `width` e `height` no cálculo de estilo.
- Se estiver 0×0, ele existe, carrega (`onLoad`), mas ocupa nada na tela.

### 2. Garanta largura e altura fixas
- Ao invés de `minWidth`/`minHeight`, use:
  ```jsx
  <img
    src={whatsappStatus.qrCode}
    alt="QR Code WhatsApp"
    width={256}
    height={256}
    style={{
      border: '4px solid red',
      display: 'block',        // evita gaps de inline
      backgroundColor: 'yellow'
    }}
  />
  ```
- Ou inline style completo:
  ```jsx
  style={{
    width: '256px',
    height: '256px',
    border: '4px solid red'
  }}
  ```

### 3. Verifique as classes do Tailwind
- Se Tailwind não estiver ativo, `w-64 h-64` não geram nada.
- Você pode remover `className` e usar apenas inline `width/height`.

### 4. Confira o valor de `src`
- No console, copie e cole o valor de `whatsappStatus.qrCode` na barra de endereços.
- Assegure que comece com `data:image/...;base64,` e termine sem truncamento.

### 5. Teste com uma imagem externa
- Substitua temporariamente:
  src="https://via.placeholder.com/256"
- Se aparecer, o problema está no seu `data URI`.

### 6. Verifique estilos de pai
- Veja se um container acima não aplica `overflow: hidden` ou `display: none`.
- Confirme também `z-index` e `position`: a imagem pode estar atrás de outro elemento.

### 7. Remova filtros CSS
- Busque por `filter: blur(...)`, `opacity: 0`, ou `visibility: hidden` no CSS global.
- Desative esses filtros para ver se a imagem reaparece.

---

Após esses ajustes, recarregue a página e verifique se a borda vermelha ou o placeholder aparecem.  
Isso garante que o QR Code seja visível assim que o `onLoad` for disparado.
